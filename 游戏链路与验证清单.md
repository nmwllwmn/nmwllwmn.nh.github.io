# 《最后一次签到》游戏链路与验证清单

## 1. 入口和全局规则

- 玩家从 `index.html` 进入，点击“进入南洄”到 `chat.html`，获得林舟的手机界面。
- 主要调查行为都应发生在手机界面内：微信、短信、备忘录、相册、邮箱、浏览器、地图、数独、密页集。
- 手机桌面 App 打开后仍在同一个浏览器标签内，不应新开标签；手机浏览器内的站点访问必须改变页面 URL，例如 `phone-browser.html?site=university`。
- 右上角“调查笔记”是独立全局功能，记录存储在 `last-checkin-investigation-notes-v1`，页面切换或重置进度后仍保留。

## 2. 进度、电量和威胁短信

电量由 `common.js` 的 `getRemainingBattery()` 根据证据和关键 flags 计算。

关键进度项：

- 解锁手机：`phoneUnlocked`
- 阅读阿远微信：`readLinzhouWechat`
- 队伍/导入提示：`readTeamDoubt` 或 `foundImportBatchHint`
- 阅读威胁短信：`readThreatSms`
- 打开相册锁：`albumLockOpened`
- 证据 A：林舟 S05-S07 为 Web Import
- 证据 B：2023 旧路线含 S08
- 证据 C：许听雨参加过 2023 试跑
- 邮箱登录：`phoneMailLoggedIn`
- NVR 入口：`foundNvrLogin`
- 证据 D：21:38 北桥监控异常
- 证据 E：IMP-2202 由沈泊执行

威胁短信规则：

- 电量高于 30%：不出现未知号码威胁短信，手机桌面短信 App 无威胁角标。
- 电量 30% 及以下：出现第一阶段未知号码短信。
- 电量 18% 及以下：威胁短信变多，并出现更强干扰。
- 电量 8% 及以下：进入最终威胁阶段。
- 已进入真相/隐藏安全状态后，不再触发新威胁：`sudokuSecretSolved`、`reportedMysteryMan` 或 `rescuedLinzhou`。

## 3. 手机内主线链路

## 4. 结局条件

### 4.1 未完成

条件：

- 证据 A-E 未齐全。

结果：

- 密页集显示“密页集未完成”。

### 4.2 普通结局：最后一次签到

条件：

- 证据 A-E 齐全。
- 未完成数独第三关，或未获得 `sudokuSecretSolved`。

结果：

- 玩家知道林舟没有完赛，也知道有人补写路线，但还不知道完整救援入口。

### 4.3 真相结局：旧站未删除

条件：

- 证据 A-E 齐全。
- 已完成数独第三关，获得 `sudokuSecretSolved`。
- 尚未从隐藏相册图片点击“记录这条线索”。

结果：

- 数独缓存恢复相册密码，玩家知道旧站、NVR 和导入任务之间存在人为串改，但还没有把隐藏照片作为报警证据提交。

### 4.4 彩蛋结局：抓住幕后黑手

条件：

- 证据 A-E 齐全。
- 已完成数独第三关：`sudokuSecretSolved`
- 已输入密码打开相册，并在隐藏相册图片点击“记录这条线索”：`albumPhotoReported` 或 `reportedMysteryMan`

结果：

- 页面标题为“结局”，不再使用“密页集”承载彩蛋结局。
- 页面自动拨打 110，显示玩家与接警中心的对话，对话会按时间一条一条出现。
- 对话说明：玩家从失踪朋友林舟的手机相册里发现隐藏照片，照片里的人在北桥雨水泵站附近，照片被单独锁住且文件名/时间像是林舟故意留下。
- 随后黑屏显示隐藏结局文字：照片里的人被带走，但林舟仍然失踪。
- 结局页“重新开始探索”不需要确认，直接清空调查进度并回到 `index.html`。

### 4.5 隐藏结局：未完成的签到

条件：

- 证据 A-E 齐全。
- 已完成数独第三关：`sudokuSecretSolved`
- 找到完整救援路线：
  - `foundB17`
  - `foundPumpStation`
  - `foundRescueWindow`

结果：

- 玩家绕开主馆，使用 B17、泵站、00:17 维护窗口救援林舟。

## 5. 分支验证清单

- 首页笔记本写入后，切换到手机、浏览器、密页集再返回首页，笔记仍保留。
- 笔记本底部有“重新开始探索”按钮，位置与复制/清空按钮隔开，点击需要确认；确认后清空调查进度但保留调查笔记。
- 手机桌面进入 App 不新开标签。
- 手机浏览器内打开官网/NVR/校园墙/网盘，均保留在手机壳中，并改变为 `phone-browser.html?site=...` URL。
- 电量高于 30% 时没有未知号码短信。
- 电量 30% 以下出现威胁短信。
- 电量 18% 以下威胁短信数量增加。
- 电量 8% 以下进入最终威胁阶段。
- 未获得数独第三关密码时，相册仍要求输入 6 位应用锁密码。
- 输入错误相册密码会提示错误；输入 `062141` 后可打开隐藏相册。
- 再次从普通入口打开相册时仍回到密码页。
- 完成数独后不会自动补齐救援路线。
- 证据齐全但无数独：普通结局。
- 证据齐全且有数独但无救援路线：真相结局。
- 证据齐全、有数独、隐藏相册图片已记录：彩蛋结局，幕后黑手被抓但林舟仍未找到。
- 证据齐全、有数独、有救援路线且未触发相册报警分支：隐藏结局。

本轮发现并修复：

- 泵站照片的“雨水泵站”原本写在 `meta` 字段，记录逻辑只检查正文，导致 `foundPumpStation` 不会设置。现已改为同时检查标题、EXIF/meta 和正文。
- 真相结局文案原本说“数独缓存恢复匿名号码与 shenbo_admin 绑定记录”，与当前数独功能不一致。现已改为“数独缓存恢复隐藏相册索引”。

- 相册三列网格在手机截图中会出现右侧卡片截断。现已改为两列自适应网格。
- 相机旧入口原本可以直接完成相册解锁。现改为进入相册密码页，避免绕过“每次打开相册都要密码”。
