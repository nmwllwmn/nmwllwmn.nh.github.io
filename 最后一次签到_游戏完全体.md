# 《最后一次签到》网页解密游戏完全体

版本：2.0  
定位：站群式网页解密 / 伪真实网站调查 / 校园悬疑 / 微恐  
参考气质：  
- 《邺山彼处》：全屏封面、玩法说明、进入后在多个网页和网站中搜索信息，强调“探寻、搜索、记录”。  
- 《清河大学》：监控系统入口、学校官网伪装、校园公告、账号登录、内部档案、多个正常网页之间互相藏线索。  

本版本推翻旧版“单一活动后台 + 按钮推进”的结构，改为“伪真实站群 + 搜索关键词 + 多账号 + 多入口碎片拼接”的网页解密。

---

## 1. 新方向总结

### 1.1 核心体验

玩家不是在一个游戏界面里点关卡，而是在一个看似真实的高校活动网络里调查：

- 活动官网
- 学校官网
- 旧论坛备份
- 校园墙
- 网盘目录
- 邮箱登录
- 监控系统
- 地图搜索
- 档案检索
- 被删除的旧网页

玩家需要像真的在上网查事一样，记下关键词、编号、时间、人名、地点和密码，把多个普通页面里的异常拼起来。

### 1.2 游戏标题

《最后一次签到》

副标题：

> 南洄大学城迹社 2026 夜行事件

### 1.3 一句话故事

朋友林舟在参加南洄大学城市夜行定向赛后失踪，活动官网却显示他已经完赛。你从他发来的最后一条消息开始，进入一组看似普通的网站，发现他真正的最后一次签到并不在官方路线里。

### 1.4 关键差异

旧版问题：

- 太像后台系统。
- 谜题太直给。
- 页面之间缺少真实网站感。
- 玩家只是按流程解锁，没有“调查”的自由感。

新版目标：

- 入口像独立网页解密游戏。
- 游戏内每个页面都像真实网站。
- 线索散落在不同网站和不同账号视角中。
- 关键词搜索是核心玩法。
- 有流程编号，但不暴露答案。
- 玩家需要做笔记。

## 2. 参考玩法原则

### 2.1 入口页原则

入口页不是直接开始解谜，而是建立气质：

- 大图或全屏视觉。
- 简短玩法说明。
- 一个明确的“进入”按钮。
- 提醒玩家需要搜索、记录、跨页面调查。
- 可以有轻微音效和黑屏过渡。

《最后一次签到》的入口按钮：

> 进入南洄

点击后进入“林舟发来的聊天页面”，而不是直接进活动后台。

### 2.2 站群原则

游戏不只有一个网站，而是一组伪网站：

| 编号 | 站点 | 表面用途 | 实际用途 |
|---|---|---|---|
| 00 | 游戏入口页 | 介绍玩法 | 建立氛围 |
| 01 | 聊天页面 | 林舟最后消息 | 给第一批关键词 |
| 02 | 夜行赛官网 | 活动介绍和成绩查询 | 第一层异常 |
| 03 | 南洄大学官网 | 新闻公告和机构页面 | 查人名、旧新闻 |
| 04 | 城迹社旧站 | 社团历史页面 | 查到 2023 试跑 |
| 05 | 校园墙 | 匿名爆料 | 查到 B17 和北桥 |
| 06 | 网盘目录 | 林舟备份文件 | 图片、PDF、录音 |
| 07 | 邮箱登录 | 多角色邮件 | 账号权限推进 |
| 08 | 监控系统 | NVR 查看器 | 关键时间线 |
| 09 | 地图搜索 | 地点索引 | 找到第 8 个点 |
| 10 | 归档终端 | 最终提交 | 结局 |

### 2.3 搜索原则

游戏内至少有 4 个搜索入口，不同搜索入口能查到不同东西：

- 活动官网搜索：查队伍、签到点、成绩。
- 大学官网搜索：查公告、人员、旧新闻。
- 校园墙搜索：查匿名帖和关键词。
- 地图搜索：查地点、路线编号。

玩家不能只靠一个搜索框通关。

### 2.4 记录原则

页面右上角放一个“小笔记”按钮，玩家可以打开记录板。  
首版可以做成本地文本框，使用 localStorage 保存。

笔记系统不自动解谜，只帮助玩家记录：

- 人名
- 地点
- 日期
- 编号
- 密码
- 异常时间

### 2.5 流程编号原则

参考《邺山彼处》的流程编号感，游戏右上角显示：

> 01 / 10

数字只表示“你已经接近第几阶段”，不告诉玩家答案。  
进入关键站点或解开关键谜题后，编号自动推进。

### 13.1.1 系列标准功能：全局笔记本

从《最后一次签到》开始，系列内所有网页解密作品都必须内置一个可随时打开的笔记本。

笔记本定位：

- 它不是提示系统。
- 它不是自动解谜面板。
- 它是玩家自己的调查记录工具。

必须支持：

- 任意页面随时打开和关闭。
- 玩家手动输入内容。
- 玩家从网页中复制内容后粘贴进去。
- 自动保存到 localStorage。
- 刷新页面、跳转页面后保留内容。
- 一键复制全部笔记。
- 一键清空笔记，清空前需要确认。
- 移动端可正常输入，不遮挡主要操作。

建议样式：

- 固定在页面右侧或右下角。
- 标题统一为“调查笔记”。
- 默认收起，避免遮挡页面。
- 打开后是一个大文本框，不做复杂富文本。

推荐占位文案：

```text
记录关键词、编号、时间、人名、密码。
例如：0606 / S04 / B17 / 21:38 / IMP-2202
```

后续系列作品必须复用同一套交互习惯，让玩家形成肌肉记忆。

## 17. 最推荐的开发策略

不要先做复杂剧情动画。  
先做“能像真的网站一样被搜索、登录、查资料”的站群骨架。

开发顺序：

2. 做入口页和聊天页。
3. 做活动官网完整第一谜题。
4. 做大学官网、旧站、校园墙三站联动。
5. 做网盘、邮箱、NVR 三个权限站点。
