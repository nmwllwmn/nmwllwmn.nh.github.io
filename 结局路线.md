## 2. 分流入口和优先级

主要入口：

- `archive.html`：结局归档页，会根据当前进度自动跳转到部分独立结局页。
- `archive.html?stay=1`：强制停留在归档页，不自动跳转，便于检查已恢复条目。
- `ending.html?type=normal`：普通结局演出页。
- `ending.html?type=truth`：真相结局演出页。
- `ending.html?type=rescue`：隐藏救援结局演出页。
- `ending.html?type=album-report`：相册报警彩蛋结局演出页。

`archive.html` 当前自动跳转规则：

| 判断顺序 | 条件 | 跳转 |
|---|---|---|
| 1 | 已满足隐藏救援结局，且未满足相册报警结局 | `ending.html?type=rescue` |
| 2 | 证据 A-E 齐全，已完成数独，未报警，且救援路线不完整 | `ending.html?type=truth` |
| 3 | 证据 A-E 齐全，未完成数独 | `ending.html?type=normal` |
| 4 | 其他情况 | 留在 `archive.html` 显示归档状态 |

注意：相册报警彩蛋结局不是由 `archive.html` 自动跳转触发，而是在隐藏照片详情页选择“报警并提交照片”后直接进入 `ending.html?type=album-report`。如果玩家已经触发相册报警，`archive.html` 会显示彩蛋结局卡片，但不会自动跳转到独立演出页。

## 3. 未完成状态：结局尚未形成

触发条件：

- 证据 A-E 没有全部齐全。

进入方式：

1. 从手机浏览器打开 `phone-browser.html?site=archive`。
2. 点击“打开备份记录”进入 `archive.html`。
3. 如果核心证据不完整，页面显示“结局尚未形成”。

页面表现：

- 已恢复条目显示为“已恢复”。
- 未恢复条目显示为“未恢复”。
- 不进入独立 `ending.html` 演出页。

验证重点：

- 任意缺少 A-E 中一个证据时，不应进入普通、真相、救援或彩蛋结局。
- `archive.html?stay=1` 也应能显示相同归档状态。

## 4. 普通结局：最后一次签到

结局含义：

玩家证明林舟没有自然完赛，成绩后半程被人为导入，但没有恢复隐藏相册/数独线索，也没有形成救援路线。

触发条件：

| 条件 | 必须状态 |
|---|---|

| `evidence.B` | true |
| `evidence.C` | true |
| `evidence.D` | true |
| `evidence.E` | true |
| `flags.sudokuSecretSolved` | false |

典型触发路线：

2. 微信/备忘录/短信/邮箱中找到线索方向。
3. 手机浏览器记录证据 A：`phone-browser.html?site=event`，保存本页记录。
4. 手机浏览器或邮箱记录证据 B：`phone-browser.html?site=oldsite` 或邮箱旧站缓存邮件。
5. 手机浏览器记录证据 C：`phone-browser.html?site=xutingyu`。
6. 校园邮箱登录 `linzhou / river17`，读取 NVR 临时账号线索，获得 `foundNvrLogin`。
7. 手机浏览器进入 NVR：`phone-browser.html?site=nvr`。
8. NVR 使用 `guest / cam042138` 登录，选择 CAM 04，时间调到 `21:38`，增强/移动画面看到 `S08 / IMP-2202`，输入 `IMP-2202` 保存证据 D。
9. 地图搜索 `IMP-2202`，展开任务详情并保存任务回执，记录证据 E。
10. 不完成数独第三关，进入 `archive.html`。

触发结果：

- `archive.html` 自动跳转到 `ending.html?type=normal`。
- 演出表现为普通结局黑屏文本：玩家被打晕，林舟没有被救出来。

验证重点：

- A-E 齐全但没有 `sudokuSecretSolved` 时必须进入普通结局。
- 不应要求 B17、泵站或 00:17。
- 不应误进入真相或救援结局。

## 5. 真相结局：旧站未删除

结局含义：

玩家已经证明成绩、旧站、NVR、导入任务之间存在人为串改，并通过数独恢复出隐藏相册入口，但没有提交隐藏照片，也没有拼出完整救援路线。

触发条件：

| 条件 | 必须状态 |
|---|---|
| `evidence.A` | true |
| `evidence.B` | true |
| `evidence.C` | true |
| `evidence.D` | true |
| `evidence.E` | true |
| `flags.sudokuSecretSolved` | true |
| `flags.albumPhotoReported` | false |
| `flags.reportedMysteryMan` | false |
| 救援路线 | 不完整，即 `foundB17`、`foundPumpStation`、`foundRescueWindow` 未全部为 true |

典型触发路线：

1. 先按普通结局路线收集证据 A-E。
2. 打开数独：`phone-game.html`。
3. 依次完成第一关、第二关、第三关。
4. 第三关提交成功后写入 `sudokuSecretSolved`，并提示相册密码 `062141`。
5. 不点击隐藏照片的“记录这条线索”。
6. 不集齐完整救援路线，尤其不要同时满足 B17、泵站、00:17。
7. 进入 `archive.html`。

触发结果：

- `archive.html` 自动跳转到 `ending.html?type=truth`。
- 演出表现为真相结局：证据链成立，但林舟仍在信号之外。

验证重点：

- 完成数独后，如果没有相册报警且救援路线不完整，应进入真相结局。
- 只找到 B17 或只找到泵站，不应进入救援结局。
- 数独只恢复隐藏相册索引和密码，不应自动补齐救援路线。

## 6. 彩蛋结局：雨夜来电 / 抓住幕后黑手

结局含义：

玩家通过数独恢复相册密码，打开隐藏照片并报警。照片里的人被警方带走，但林舟仍未找到。

触发条件：

| 条件 | 必须状态 |
|---|---|
| `sudokuSecretSolved` | true，必须先恢复隐藏相册索引 |
| `albumLockOpened` | true |
| `albumPhotoReported` 或 `reportedMysteryMan` | true |
| 归档显示彩蛋结局 | 还需要证据 A-E 齐全 |

当前代码说明：

- `phone-gallery.html` 会同时校验相册密码 `062141` 和 `sudokuSecretSolved`。
- 玩家必须先通过数独第三关恢复隐藏相册索引，再用 `062141` 打开相册。
- 点击隐藏照片详情页的“处理这张照片”会先写入 `foundB17`、`foundPumpStation`、`keptAlbumPhoto`，然后出现两个选择。
- 选择“报警并提交照片”会写入 `reportedMysteryMan`、`albumPhotoReported`，随后跳转到 `ending.html?type=album-report`。
- 选择“暂不报警，继续找人”不会写入报警 flag，可以继续走隐藏救援线。

典型触发路线：

1. 完成数独三关，获得相册密码 `062141`。
2. 打开相册：`phone-gallery.html`。
3. 输入 `062141`，写入 `albumLockOpened`，进入 `phone-gallery.html?view=grid`。
4. 点击唯一照片进入 `phone-gallery.html?view=photo`。
5. 点击照片详情里的“处理这张照片”。
6. 选择“报警并提交照片”，页面提示保留原图并自动拨打 110。

触发结果：

- 直接进入 `ending.html?type=album-report`。
- `ending.html` 写入 `reportedMysteryMan` 和 `albumPhotoReported`。
- 演出表现为 110 接警对话，最终黑屏显示照片里的人被警方带走，但林舟仍然失踪。

与救援结局的关系：

- 处理相册照片会写入 `foundB17` 和 `foundPumpStation`，但不会写入 `foundRescueWindow`。
- 不报警不会触发彩蛋报警结局，也不会阻断隐藏救援线。
- 报警会触发彩蛋报警结局，并使 `archive.html` 不再自动跳转救援演出。
- 如果报警前已经找到 `foundRescueWindow`，从归档状态看也可能具备救援路线，但相册报警的独立演出已经先发生。
- 当前 `archive.html` 的自动跳转和归档正文都以“已报警状态不再进入救援演出”为准。

验证重点：

- 每次从普通入口打开相册应先显示密码页。
- 输入错误密码应停留在锁定页。
- 未完成数独时，即使输入 `062141` 也不应打开隐藏相册。
- 输入 `062141` 后才能进入隐藏相册。
- 点击处理照片后应出现“报警并提交照片”和“暂不报警，继续找人”两个选择。
- 选择不报警后应留在相册页，并且救援线仍可触发。
- 选择报警后应直接进入 `ending.html?type=album-report`。

典型触发路线：

2. 完成数独第三关，获得 `sudokuSecretSolved`。
3. 找到 B17：
   - 手机浏览器 `phone-browser.html?site=wall` 保存校园墙记录，或

   - 手机浏览器 `phone-browser.html?site=drive` 保存 B17 备份目录，或
   - 地图 `phone-map.html?place=pump` 保存地点记录。
5. 找到 00:17 维护窗口：
   - 手机浏览器 `phone-browser.html?site=b17` 保存 B17 门禁掉线记录。

触发结果：

- `archive.html` 自动跳转到 `ending.html?type=rescue`。
- `ending.html?type=rescue` 写入 `rescuedLinzhou`。

验证重点：

- 缺少 `foundRescueWindow` 时不能进入救援结局。
- 缺少 `sudokuSecretSolved` 时即使 B17、泵站、00:17 都已找到，也不能进入救援结局。
- 已触发相册报警后，`archive.html` 不会自动跳转救援演出。

## 8. 结局路线推荐验收顺序

建议每次改结局条件、证据记录或相册/NVR/地图时，从清空 `localStorage` 开始按以下顺序验收：

1. 未完成状态：只拿 1-4 条证据，进入 `archive.html`，确认不跳转。
2. 普通结局：拿齐 A-E，不完成数独，进入 `archive.html`，确认跳 `ending.html?type=normal`。
3. 真相结局：A-E + 数独，不报警，不齐救援路线，确认跳 `ending.html?type=truth`。
4. 彩蛋结局：完成数独，打开相册，处理隐藏照片并选择报警，确认跳 `ending.html?type=album-report`。
5. 隐藏救援：A-E + 数独 + B17 + 泵站 + 00:17，不报警或在相册选择暂不报警，确认跳 `ending.html?type=rescue`。

## 9. 当前实现中的注意点

- `ending.html` 不再接受空 `type` 或不满足条件的直接访问，会回到 `archive.html?stay=1`。
- 相册密码 `062141` 必须在 `sudokuSecretSolved` 后才能生效。
- 旧版状态里如果只有 `albumLockOpened`、没有 `sudokuSecretSolved`，直接访问相册详情也会回到锁定页。
- 相册照片会设置 B17 和泵站，但只有选择报警才会设置报警 flags；两种选择都不设置 00:17。
- 救援结局自动跳转要求未触发相册报警；如果后续希望“报警后仍可救援”，需要调整 `archive.html` 的优先级。
- 旧页面 `event.html`、`club2023.html`、`mail.html`、`map.html`、`nvr.html` 仍能写入部分旧 flag/evidence，但当前主体验应以 `phone-*.html` 和 `phone-browser.html?site=...` 为准。
